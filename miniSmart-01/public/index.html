miniSmarts V0.1

miniSmart-01

Telegram Gateway oficial del ecosistema miniSmarts, orientado a modelos pequeños (3B+) y entornos de recursos limitados.

Rol y alcance

  • Mini: miniSmart-01
  • Tipo: gateway
  • Puerto: 7000
  • Namespace: /api/01

Endpoints API públicos

  • GET /api/01/capabilities
  • GET /api/01/contract
  • POST /api/01/send
  • GET /api/01/health
  • GET /api/01/docs

CLI OPERATIONS

  • Ciclo de vida del servicio: install, uninstall, start, stop, restart, status, logs, docs con mini 01.
  • Gestión de token: minismarts token set 01, minismarts token get 01, minismarts token validate 01.
  • Ubicación del archivo de entorno: /etc/minismarts/01.env.
  • Cómo validar bot: ejecutar minismarts token validate 01 para confirmar acceso a Telegram con getMe.

Flujo de instalación real

  • Instalar con npm install -g minismarts-01.
  • Ejecutar minismarts install 01.
  • Si falta entorno, se solicita token, se valida y se persiste en /etc/minismarts/01.env.
  • Se instala y arranca miniSmart-01.service.

Publicación npm

  • Validar paquete con npm pack.
  • Autenticarse con npm login.
  • Publicar con npm publish.
